# FinSight AI: Comprehensive Financial Intelligence Platform

FinSight AI is an advanced, full-stack data science platform designed to aggregate, analyze, and visualize global macroeconomic trends, market sentiment, and sector-specific financial intelligence. By leveraging natural language processing, machine learning models, and interactive data visualization, FinSight AI transforms raw financial data into actionable, easy-to-understand insights.

This document serves as the central guide for the entire project, detailing the architecture, components, and instructions for setting up the data environment.

## Project Architecture

The FinSight AI repository is structured into five core components, each serving a distinct phase of the data science lifecycle:

### 1. Data (`/data`)
The data directory acts as the central storage for all datasets moving through the pipeline. It is strictly organized to maintain data integrity and reproducibility:
*   **Raw**: Unmodified datasets ingested directly from external financial APIs and news sources.
*   **Interim**: Data that has been cleaned and partially processed but is not yet ready for model training.
*   **Processed**: Fully standardized datasets ready for machine learning algorithms.
*   **Specialized Directories**: Includes folders for `embeddings`, `entities`, `classification`, `clusters`, and `topics`, storing the output data generated by the natural language processing models.

### 2. Notebooks (`/notebooks`)
The notebooks directory contains Jupyter Notebooks used for research, prototyping, and exploratory data analysis. 
*   This is where the initial data exploration occurs.
*   Data scientists use this space to test feature engineering techniques, evaluate early model performance, and document the rationale behind algorithmic choices before moving code into production scripts.

### 3. Models (`/models`)
This directory contains the serialized versions of our trained machine learning and natural language processing models.
*   It includes the core artificial intelligence engine used for sentiment analysis, entity extraction, and financial text classification.
*   Models are version-controlled here so that the dashboard can load them rapidly for real-time inference without retraining.

### 4. Reports (`/reports`)
The reports directory is designated for static outputs, such as data quality reports, model performance metrics, and automated PDF summaries.
*   It serves as a historical record of model accuracy over time.
*   It includes data profiling reports that highlight anomalies or missing values in the raw datasets.

### 5. Dashboard Visualization (`/DashBoard`)
The Dashboard is the interactive, user-facing front-end of the FinSight AI platform. Built with Streamlit and styled with a premium, modern aesthetic, it allows end-users to interact with the data and models seamlessly.
*   **Executive Command Center**: A high-level overview of market pulse, system status, and aggregate sentiment.
*   **News Explorer**: An interactive module to filter and search through processed financial news articles, displaying model-generated confidence scores.
*   **Sector Intelligence**: Visualizes risk scores, growth metrics, and recommendations aggregated by financial sectors.
*   **Templating Engine**: The dashboard utilizes a custom Python-to-HTML templating system, ensuring that the complex backend logic is cleanly separated from the user interface styling.

## Dataset Download and Installation Guide

To run FinSight AI locally, you must first provision the necessary datasets. Follow these detailed steps to download and install the required data.

### Step 1: Environment Preparation
Ensure you have Python installed on your system. It is highly recommended to use a virtual environment.
Navigate to the root directory of the project and install the required dependencies:
`pip install -r DashBoard/requirements.txt`

### Step 2: Acquiring the Raw Data
The project relies on historical financial news and market data. 
1.  Navigate to your designated financial data provider or open-source repository.
2.  Download the raw CSV or JSON data files to your local machine.

### Step 3: Installing the Data into the Project
Once the data is downloaded, it must be placed into the correct directories so the pipeline can process it.
1.  Extract the downloaded files if they are compressed.
2.  Copy the uncompressed raw data files into the `FinSight_AI/data/raw/` directory.

### Step 4: Running the Processing Pipeline
The raw data must be cleaned and transformed into the `processed` format required by the models and the dashboard.
1.  Open your terminal or command prompt.
2.  Navigate to the `notebooks` directory and execute the data preparation notebook.
3.  The logic will automatically ingest the files from `data/raw/`, perform necessary cleaning, generate numerical embeddings, and save the finalized datasets into `data/processed/`.

### Step 5: Launching the Dashboard
Once the processed data is in place, you can launch the visualization interface.
1.  Open your terminal.
2.  Navigate specifically into the `DashBoard` directory:
    `cd DashBoard`
3.  Launch the application using the Streamlit framework:
    `python -m streamlit run app.py`
4.  The application will start, and your default web browser will open the FinSight AI dashboard.
